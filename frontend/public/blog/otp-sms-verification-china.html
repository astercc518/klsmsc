OTP SMS Verification for China: A Practical Guide for App Developers

📅 May 2025⏱ 11 min read🏷 OTP · China · Verification · SMS

Chinese mobile number verification is one of the most technically demanding SMS use cases globally. This guide walks through exactly what it takes to achieve 95%+ delivery rates on +86 numbers.

Why OTP Delivery to China Is Uniquely Challenging

China's mobile infrastructure filters a significant portion of international SMS before it ever reaches the end user. Unlike domestic SMS (sent within China), international SMS to +86 numbers passes through carrier-controlled ingress gateways that apply multiple layers of filtering:

  1. Source filtering: Is the sending gateway whitelisted with this carrier?
  2. Volume filtering: Is this source sending at unusual rates?
  3. Content filtering: Does the message content match spam/phishing patterns?
  4. Sender ID filtering: Is the sender ID on the allowed list?

Messages failing any check are silently dropped — no error is returned to the sending gateway in many cases, so you may think messages are delivering when they aren't.

The Perfect OTP Message Format for China

Message format significantly affects delivery rate to Chinese numbers. Follow this template:

【YourApp】Your verification code is 847291.
Valid for 10 minutes. Do not share this code with anyone.

Characters to Avoid in China OTP Messages

  • URLs of any kind (http://, https://, shortened URLs)
  • Emoji or special Unicode characters
  • Financial keywords: 贷款 (loan), 投资 (investment), 理财 (wealth management)
  • Phrases resembling prize/lottery notifications
  • Contact information (phone numbers, email addresses) other than the OTP code itself

Code Example: OTP Send with DLR Polling

# Python example
import asyncio
import httpx

async def send_otp_china(phone: str, code: str) -> bool:
    """Send OTP to Chinese number with DLR polling."""
    payload = {
        "to": phone,
        "text": f"【MyApp】Your code is {code}. Valid 10 min.",
        "type": "otp",
        "callback_url": "https://api.myapp.com/sms/dlr"
    }

    async with httpx.AsyncClient() as client:
        # Primary route
        resp = await client.post(
            "https://api.kaolach.com/v1/messages",
            json=payload,
            headers={"Authorization": f"Bearer {API_KEY}"}
        )
        msg = resp.json()

        # Poll for DLR up to 30 seconds
        for _ in range(6):
            await asyncio.sleep(5)
            status = await get_message_status(client, msg["message_id"])
            if status == "delivered":
                return True
            if status in ["failed", "rejected"]:
                break

        # Fallback to secondary route
        return await send_via_backup_route(phone, code)

Rate Limiting to Prevent SMS Pumping

SMS pumping fraud — where bots trigger massive OTP requests to premium-rate numbers — can drain thousands of dollars in minutes. Implement these defenses:

# Rate limiting rules (implement in Redis or similar)
MAX_OTP_PER_PHONE = 3        # Per phone number per hour
MAX_OTP_PER_IP = 10          # Per IP address per hour
MAX_OTP_PER_USER = 5         # Per authenticated user per day
SUSPICIOUS_THRESHOLD = 50    # Alert if >50 unique phones in 10 min

Handling Edge Cases

Hong Kong vs Mainland China

Don't confuse Hong Kong (+852) with Mainland China (+86). They are different carrier networks:

  • +86 — Mainland China: China Mobile, China Unicom, China Telecom
  • +852 — Hong Kong: HKT, HKBN, SmarTone, CSL (standard international routing applies, much simpler)
  • +853 — Macau: CTM, 3 Macau

Number Portability in China

China implemented number portability in 2019. This means a +86 number starting with 134 (historically China Mobile) may now be on China Unicom or China Telecom. If your gateway hard-routes by number prefix, you may be routing to the wrong carrier after portability. Use a gateway that performs real-time HLR lookups for China numbers.

Virtual Numbers (VoIP Lines)

Some Chinese users register with virtual/VoIP phone numbers from services like Google Voice China alternatives. SMS delivery to these numbers is unreliable — filter them out using number validation before sending.
