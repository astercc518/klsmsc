# SMS Gateway 管理员系统维护手册

> **文档版本**：v1.0  
> **适用对象**：系统管理员、运维、技术负责人  
> **说明**：命令与路径以仓库默认 **Docker Compose** 部署为例，生产环境请替换为实际主机名、密码与卷路径。

---

## 1. 系统架构概览

典型组件：

| 组件 | 作用 |
|------|------|
| **api** | FastAPI 后端，对外 HTTP API 与管理接口 |
| **frontend** | Vue 静态站点 / Nginx |
| **mysql** | 业务数据持久化 |
| **redis** | 缓存、限流、分布式锁等 |
| **rabbitmq** | Celery 消息队列 |
| **worker** / **worker-sms** | Celery 异步任务（短信发送、批量、Webhook、DLR 等） |

编排文件：项目根目录 `docker-compose.yml`。

---

## 2. 日常启停与状态

```bash
cd /var/smsc

# 启动全部服务（后台）
docker compose up -d

# 查看容器状态
docker compose ps

# 查看 API 日志
docker logs -f smsc-api --tail 200

# 查看短信 Worker 日志
docker logs -f smsc-worker-sms --tail 200

# 重启单个服务（例：API）
docker compose restart api
```

**镜像更新后**：

```bash
docker compose build api frontend worker worker-sms
docker compose up -d api frontend worker worker-sms
```

## 3. 环境变量与密钥

- 数据库、Redis、RabbitMQ 等连接信息在 **`.env`** 或 `docker-compose.yml` 的 `environment` 中配置。  
- **勿将 `.env` 提交到 Git**；生产环境使用密钥管理或部署平台注入。  
- JWT、DLR 回调 Token、Telegram Bot Token 等在 **系统配置** 页面或环境变量中维护，修改后通常需 **重启 API / Worker** 生效（视项而定）。

---

## 4. 数据库与迁移

- 应用启动时 API 容器常执行 **`alembic upgrade head`**（见镜像入口脚本），确保迁移已提交到仓库。  
- 手动连接 MySQL（示例，密码见 `.env`）：

```bash
docker exec -it smsc-mysql mysql -uroot -p sms_system
```

- 重大变更前务必 **全量备份**：

```bash
docker exec smsc-mysql mysqldump -uroot -p'YOUR_ROOT_PASSWORD' sms_system > backup_$(date +%F).sql
```

## 6. Celery Worker 与 SMPP

- **短信 Worker** 建议 **并发数保持较低**（如 1），避免同一上游 SMPP 账号多进程重复绑定。  
- 环境变量示例（见 `docker-compose.yml`）：`SMPP_SUBMIT_RESP_WAIT_SECONDS`、`SMPP_DLR_SOCKET_HOLD_SECONDS`、`DLR_SENT_TIMEOUT_HOURS` 等。  
- 修改后 **重启 worker-sms**。

更细的 DLR、HTTP 回调与排错可参考仓库内文档：

- `docs/SMPP短信回执对接指南.md`  
- `docs/DLR_TROUBLESHOOTING.md`

## 7. 知识库与静态资源

- 知识库附件物理路径（容器内）：`/app/data/knowledge/{文章ID}/{uuid}_{文件名}`。  
- 宿主机应对应挂载 **同一数据卷**，否则上传/下载不一致。  
- PDF 可由脚本生成：`python3 backend/scripts/md2pdf.py 输入.md 输出.pdf`

## 8. 监控与日志

- API / Worker 日志：`docker logs`。  
- 应用内日志目录（视挂载）：`backend/logs` 或容器内 `/app/logs`。  
- MySQL 慢查询、Redis 内存、磁盘空间需纳入常规巡检。

---

## 9. 常见故障速查

| 现象 | 排查方向 |
|------|----------|
| 全站 502 / 无法访问 | 反向代理、API 容器是否运行、`docker compose ps` |
| 短信积压不发 | `worker-sms` 是否运行、RabbitMQ 是否可达、Redis 是否阻塞 |
| 大面积认证失败 | JWT 密钥是否被轮换、系统时间是否一致 |
| DLR 不到账 | 上游回调 URL、Token、白名单；参见 DLR 文档 |

---

## 10. 安全与合规

- 定期更新依赖与基础镜像，修补高危 CVE。  
- 限制管理后台 **公网暴露**，建议使用 VPN 或 IP 白名单。  
- 审计敏感操作（改价、调账、删数）并保留日志。
