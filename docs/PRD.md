## 1. 产品概述

### 1.1 产品定位
面向企业级客户的国际短信发送平台，提供灵活的多通道路由、差异化计费和品牌化发送能力。

### 1.2 产品愿景
成为最可靠、最灵活、最智能的国际短信服务平台。

### 1.3 目标用户画像

**用户群体1: 出海互联网企业**
- 需求: 向海外用户发送验证码、通知
- 痛点: 送达率低、成本高、缺乏技术支持
- 期望: 稳定可靠、价格透明、技术文档完善

**用户群体2: 电商平台**
- 需求: 订单通知、物流提醒
- 痛点: 高峰期延迟、通道单一
- 期望: 高并发处理、多通道备份

**用户群体3: 金融科技公司**
- 需求: 交易验证码、风控通知
- 痛点: 对安全性和可靠性要求极高
- 期望: 99.9%可用性、端到端加密

**用户群体4: 营销服务商**
- 需求: 批量营销短信
- 痛点: 成本敏感、需要自定义品牌
- 期望: 成本优化路由、自定义发送方ID

## 2. 核心功能需求

### 2.1 短信发送功能

#### 2.1.1 单条发送
**用户故事**: 
> 作为开发者，我希望能够通过简单的API调用发送一条短信，以便快速集成到我的应用中。

**功能描述**:
- 支持E.164格式的国际号码
- 支持纯文本和Unicode内容
- 自动识别目标国家和运营商
- 自动选择最优通道
- 实时返回发送结果

**接受标准**:
- [ ] API响应时间 < 200ms
- [ ] 支持至少160个字符（GSM-7）
- [ ] 支持中文和特殊字符（UCS-2）
- [ ] 返回唯一消息ID用于追踪

**优先级**: P0（最高）

#### 2.1.2 批量发送
**用户故事**:
> 作为营销人员，我希望能够一次性向10,000个用户发送促销短信，而不需要逐条调用API。

**功能描述**:
- 支持批量上传号码列表
- 支持CSV文件导入
- 支持变量替换（个性化内容）
- 异步处理，返回批次ID
- 实时查询批次进度

**接受标准**:
- [ ] 单批次支持10,000+号码
- [ ] 支持模板变量（如：{name}, {code}）
- [ ] 提供批次状态查询接口
- [ ] 支持导出发送结果

**优先级**: P1（高）

#### 2.1.3 定时发送
**用户故事**:
> 作为运营人员，我希望能够在特定时间发送短信（如用户当地时间早上9点），以提高打开率。

**功能描述**:
- 支持指定发送时间
- 支持按时区调整
- 支持取消定时任务

**接受标准**:
- [ ] 时间精度到分钟
- [ ] 支持时区转换
- [ ] 提供取消接口

**优先级**: P2（中）

### 2.2 智能路由功能

#### 2.2.1 自动路由
**用户故事**:
> 作为系统管理员，我希望系统能够根据国家、成本、质量自动选择最优通道，而无需手动配置每一条短信。

**功能描述**:
- 自动识别目标国家代码
- 根据路由规则选择通道
- 支持多种路由策略（优先级、成本、质量、负载均衡）
- 故障自动切换

**路由策略**:
1. **优先级路由**: 按通道优先级排序选择
2. **成本优先**: 选择价格最低的通道
3. **质量优先**: 选择送达率最高的通道
4. **负载均衡**: 按权重分配流量

**接受标准**:
- [ ] 路由决策时间 < 50ms
- [ ] 支持实时切换路由策略
- [ ] 故障通道自动屏蔽
- [ ] 记录路由决策日志

**优先级**: P0（最高）

#### 2.2.2 指定通道发送
**用户故事**:
> 作为企业客户，我希望能够指定特定通道发送短信，以满足特殊业务需求。

**功能描述**:
- 支持API参数指定通道
- 验证通道是否支持目标国家
- 通道不可用时返回错误

**接受标准**:
- [ ] 支持通过channel_code指定通道
- [ ] 通道不可用时给出明确错误提示

**优先级**: P1（高）

### 2.3 多国家计费功能 ⭐

#### 2.3.1 差异化计费
**用户故事**:
> 作为财务人员，我希望系统能够根据不同国家、不同运营商、不同通道自动计算准确的费用。

**功能描述**:
- 支持国家级计费规则
- 支持运营商级计费规则（MCC/MNC）
- 支持通道级计费规则
- 长短信自动拆分计费
- 支持多币种（USD/CNY/EUR）

**计费规则优先级**:
```
通道+国家+运营商价格
    ↓ (不存在)
通道+国家价格
    ↓ (不存在)
通道默认价格
    ↓ (不存在)
返回错误
```

**计费场景示例**:
1. 发送到中国移动：通道A = ¥0.05/条，通道B = ¥0.04/条 → 自动选通道B
2. 发送到美国：通道A支持 = $0.008/条，通道B不支持 → 选通道A
3. 长短信（200字符）：自动拆分为2条 × 单价

**接受标准**:
- [ ] 计费精确到小数点后4位
- [ ] 长短信计费符合行业标准（GSM-7: 160/153, UCS-2: 70/67）
- [ ] 支持至少3种币种
- [ ] 提供计费预估API
- [ ] 发送失败不扣费

**优先级**: P0（最高）

#### 2.3.2 计费查询和报表
**用户故事**:
> 作为账户管理员，我希望能够查看详细的计费明细和统计报表，以便财务对账。

**功能描述**:

- 导出账单（Excel/CSV/PDF）

**接受标准**:
- [ ] 支持按多维度查询

**优先级**: P1（高）

### 2.4 发送方ID管理 ⭐

#### 2.4.1 自定义发送方ID
**用户故事**:
> 作为品牌方，我希望短信显示我的品牌名称（如"Alibaba"），而不是一个陌生号码，以提高用户信任度。

**功能描述**:
- 支持申请自定义发送方ID
- 支持字母和数字格式
- 按国家审核（某些国家需要注册）
- 支持为不同国家配置不同SID
- 支持为不同通道配置不同SID

**SID类型**:
- **数字SID**: 最长15位，全球通用
- **字母SID**: 最长11位，部分国家支持
- **短代码**: 特定国家的专用号码（如美国5位短代码）

**各国SID要求示例**:
- 🇺🇸 美国: 支持字母SID，需预注册
- 🇨🇳 中国: 仅支持数字SID
- 🇮🇳 印度: 需要6位模板ID（DLT注册）
- 🇬🇧 英国: 支持字母SID，11位限制

**接受标准**:
- [ ] 支持在线申请SID
- [ ] 审核工作流（待审核→已通过→已激活）
- [ ] SID状态管理（启用/禁用/冻结）
- [ ] 违规SID自动屏蔽

**优先级**: P0（最高）

#### 2.4.2 可选SID发送
**用户故事**:
> 作为开发者，我希望在发送短信时可以选择使用哪个发送方ID，或让系统自动选择最合适的。

**功能描述**:
- 支持API参数指定SID
- 根据账户、国家、通道自动匹配SID
- 支持设置默认SID
- SID不可用时使用备用SID

**SID选择逻辑**:
```
1. 用户API指定的SID
   ↓ (验证权限和可用性)
   成功 → 使用该SID
   失败 ↓
2. 账户+国家+通道专属SID
   ↓ (查询数据库)
   存在 → 使用该SID
   不存在 ↓
3. 账户+通道通用SID
   ↓ (查询数据库)
   存在 → 使用该SID
   不存在 ↓
4. 通道默认SID
   ↓ (通道配置)
   存在 → 使用该SID
   不存在 ↓
5. 返回错误：无可用SID
```

**接受标准**:
- [ ] SID匹配时间 < 20ms
- [ ] 支持回退机制
- [ ] 记录SID选择日志
- [ ] 用户指定的SID必须验证权限

**优先级**: P0（最高）

### 2.5 状态追踪功能

#### 2.5.1 消息状态查询
**用户故事**:
> 作为开发者，我希望能够查询短信的实时状态，以便向用户展示发送进度。

**功能描述**:
- 通过消息ID查询状态
- 支持批量查询
- 实时状态更新

**状态定义**:
- `pending`: 待发送（已接收请求）
- `queued`: 已入队（等待Worker处理）
- `sent`: 已发送到通道
- `delivered`: 已送达用户
- `failed`: 发送失败
- `expired`: 超时未送达
- `rejected`: 被拒绝（号码无效、黑名单等）

**状态流转**:
```
pending → queued → sent → delivered ✅
                     ↓
                  failed ❌
                     ↓
                  expired ⏱️
```

**接受标准**:
- [ ] 查询响应时间 < 100ms
- [ ] 状态更新延迟 < 30秒
- [ ] 支持批量查询（最多100条）

**优先级**: P0（最高）

#### 2.5.2 状态回调（Webhook）
**用户故事**:
> 作为开发者，我希望短信状态变化时系统能主动通知我，而不是轮询查询。

**功能描述**:
- 支持配置回调URL
- 状态变更时HTTP POST通知
- 支持签名验证（防伪造）
- 失败自动重试

**回调触发时机**:
- 发送成功时（sent）
- 送达成功时（delivered）
- 发送失败时（failed）

**回调格式**:
```json
POST {callback_url}
Content-Type: application/json
X-Signature: sha256_hmac_signature

{
  "message_id": "MSG1735577200ABC12345",
  "status": "delivered",
  "phone_number": "+8613800138000",
  "country_code": "CN",
  "submit_time": "2025-12-30T15:30:00Z",
  "delivery_time": "2025-12-30T15:30:05Z",
  "error_code": null,
  "timestamp": "2025-12-30T15:30:05Z"
}
```

**接受标准**:
- [ ] 回调延迟 < 5秒
- [ ] 失败重试3次（1分钟、5分钟、15分钟）
- [ ] 提供HMAC-SHA256签名验证
- [ ] 记录回调日志

**优先级**: P1（高）

## 3. 非功能需求

### 3.1 性能需求

#### 3.1.1 响应时间
- API响应时间 P95 < 200ms
- API响应时间 P99 < 500ms
- 后台页面加载时间 < 2秒
- 数据库查询时间 < 100ms

#### 3.1.2 吞吐量
- 支持10,000 TPS（每秒事务数）
- 支持百万级/日的发送量
- 支持峰值流量（平时的5-10倍）

#### 3.1.3 并发数
- 支持10,000+并发连接
- 单通道支持1,000 TPS

### 3.2 可用性需求
- **系统可用性**: 99.9%（年宕机时间 < 8.76小时）
- **数据可靠性**: 99.99%（消息不丢失）
- **故障恢复**: RTO < 15分钟，RPO < 5分钟

### 3.3 安全需求
- API密钥认证（HMAC-SHA256签名）
- HTTPS强制加密
- IP白名单限制
- 请求频率限制（Rate Limiting）
- 数据库密码加密存储
- 敏感信息脱敏展示
- 审计日志记录（所有操作可追溯）

### 3.4 易用性需求
- API设计符合RESTful规范
- 完善的API文档（Swagger/OpenAPI）
- SDK支持主流语言（Python/Java/PHP/Node.js）
- 管理后台操作简单直观
- 提供快速接入指南（5分钟完成集成）

### 3.5 可扩展性需求
- 微服务架构
- 水平扩展能力
- 数据库分片支持
- 新通道快速接入（< 1天）
- 新国家快速支持（< 1小时）

## 4. 用户流程

### 4.1 开发者首次接入流程
```
1. 注册账号
   ↓
2. 登录管理后台
   ↓
3. 获取API密钥
   ↓
4. 充值账户余额
   ↓
5. （可选）申请发送方ID
   ↓
6. 阅读API文档
   ↓
7. 集成API到应用
   ↓
8. 发送测试短信
   ↓
9. 验证接收
   ↓
10. 正式使用
```

### 4.2 发送短信完整流程
```
【客户端】
开发者调用API
  POST /api/v1/sms/send
   ↓
【API网关】
1. 验证API密钥
2. 检查IP白名单
3. 请求限流检查
   ↓
【API服务】
4. 解析请求参数
5. 验证手机号格式
6. 识别国家代码（+86 → CN）
   ↓
【路由引擎】
7. 查询可用通道列表
8. 根据策略选择最优通道
   ↓
【SID管理器】
9. 匹配发送方ID
   ↓
【计费引擎】
10. 查询价格（通道+国家）
11. 计算短信条数
12. 计算总费用
13. 检查余额
14. 扣减余额
   ↓
【消息队列】
15. 消息入队（RabbitMQ）
16. 返回消息ID给客户端
   ↓
【通道Worker】
17. 消费队列消息
18. 调用第三方通道API
19. 更新状态为 sent
   ↓
【第三方通道】
20. 发送短信到用户手机
21. 返回送达回执
   ↓
【状态追踪】
22. 更新状态为 delivered
23. 触发Webhook回调
   ↓
【客户端】
24. 接收回调通知
25. 更新业务状态
```

### 4.3 管理员配置通道流程
```
1. 登录管理后台
   ↓
2. 进入通道管理页面
   ↓
3. 点击"添加通道"
   ↓
4. 填写通道信息
   - 通道名称
   - 协议类型（SMPP/HTTP）
   - 连接信息（IP、端口、账号、密码）
   - 优先级、权重、最大TPS
   ↓
5. 测试连接
   ↓ (成功)
6. 保存通道
   ↓
7. 配置路由规则
   - 选择支持的国家
   - 设置优先级
   ↓
8. 配置计费规则
   - 按国家设置价格
   - 设置生效日期
   ↓
9. 启用通道
   ↓
10. 完成
```

## 7. 附录

### 7.1 术语表
- **SMPP**: Short Message Peer-to-Peer，短信点对点协议
- **TPS**: Transactions Per Second，每秒事务数
- **MCC**: Mobile Country Code，移动国家代码
- **MNC**: Mobile Network Code，移动网络代码
- **E.164**: 国际电话号码格式标准（如+8613800138000）
- **GSM-7**: GSM 7位编码，用于英文短信（160字符/条）
- **UCS-2**: Unicode字符集，用于多语言短信（70字符/条）
- **SID**: Sender ID，发送方ID，短信显示的发送方标识
- **DLT**: Distributed Ledger Technology（印度短信注册系统）
- **Webhook**: HTTP回调通知机制
