# SMSC 项目备份与迁移指南

## 目标服务器信息

- **地址**: 107.148.34.177
- **SSH 端口**: 31837
- **用户**: root
- **密码**: （请妥善保管，通过 MIGRATE_PASS 环境变量传入）

## 迁移步骤

### 一、在当前服务器执行备份

```bash
cd /var/smsc
chmod +x scripts/backup_and_migrate.sh
./scripts/backup_and_migrate.sh backup
```

备份将生成：
- `.backup_migrate/sms_system.sql` - 数据库导出
- `.backup_migrate/smsc_backup_YYYYMMDD_HHMMSS.tar.gz` - 项目压缩包
- `.backup_migrate/.env` - 环境配置（若存在）
- `.backup_migrate/restore_on_new_server.sh` - 新服务器一键恢复脚本

### 二、传输到新服务器

```bash
# 方式1：使用环境变量（密码含特殊字符请用单引号）
MIGRATE_HOST=107.148.34.177 MIGRATE_PORT=31837 MIGRATE_USER=root \
MIGRATE_PASS='你的密码' ./scripts/backup_and_migrate.sh transfer

# 方式2：已配置 SSH 密钥时可省略密码
MIGRATE_HOST=107.148.34.177 MIGRATE_PORT=31837 MIGRATE_USER=root \
./scripts/backup_and_migrate.sh transfer
```

需安装 `sshpass`（若使用密码）：`apt install sshpass`

### 三、在新服务器上恢复

1. SSH 登录新服务器：

```bash
ssh -p 31837 root@107.148.34.177
```

2. 确保已安装 Docker 和 Docker Compose

3. 创建项目目录并进入：

```bash
mkdir -p /var/smsc
cd /var/smsc
```

4. 将传输过来的 `.backup_migrate` 目录放到 `/var/smsc/` 下（若 transfer 已执行，应已存在）

5. 执行一键恢复：

```bash
chmod +x .backup_migrate/restore_on_new_server.sh
./.backup_migrate/restore_on_new_server.sh
```

或使用迁移脚本的 restore 子命令（若项目已解压）：

```bash
./scripts/backup_and_migrate.sh restore
```

### 四、验证

- 前端: http://107.148.34.177:80
- API: http://107.148.34.177:8000
- RabbitMQ 管理: http://107.148.34.177:15672 (guest/guest)

## 注意事项

1. **防火墙**：确保新服务器开放 80、8000、3306、6379、5672、15672 等端口
2. **.env**：若备份时无 .env，需在新服务器手动创建并配置（参考 .env.example）
3. **Telegram Bot**：迁移后需确认 TELEGRAM_BOT_TOKEN 等配置正确
4. **SSL 证书**：certs/ 目录未包含在备份中，若需 HTTPS 请单独配置
